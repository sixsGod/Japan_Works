## PowerShell 核心特性
PowerShell 是面向对象的，这意味着它处理的不是简单的文本，而是 .NET 对象。可以创建自定义 cmdlet（命令）和模块
1. 面向对象 - 命令输出的是对象（如进程、文件），而非纯文本。
2. 管道（Pipeline） - 支持对象传递，可链式处理数据。
3. 统一命名规范 - 采用 动词-名词 格式（如 Get-Process）。
4. 跨平台支持 - 可在 Windows、Linux、macOS 运行（PowerShell Core）。
5. 深度集成 .NET - 可直接调用 .NET 类库和方法。
6. 模块化设计 - 功能以模块形式扩展（如 ActiveDirectory 模块）。
7. 自动化脚本 - 支持复杂脚本编写（变量、循环、函数等）。
8. 远程管理 - 通过 Enter-PSSession 或 Invoke-Command 管理远程机器。
9. 别名系统 - 支持简写（如 ls 替代 Get-ChildItem）。
10. 强大的帮助系统 - 内置 Get-Help 和示例代码。

## PowerShell 核心概念
查看 PowerShell 版本 :$PSVersionTable
- Cmdlet：最小命令单元
  + 基于 .NET 的类，运行后会返回一个或多个 .NET 对象。命名规则：动词-名词
  + 查看所有可用的 Cmdlet：Get-Command -CommandType Cmdlet
- 对象管道（Object Pipeline）
  + 在 PowerShell 中，管道传递的是完整的 .NET 对象。
- Provider 和 PSDrive：资源驱动器抽象
  + 常见 Provider 类型

  |Provider|说明|
  |-|-|
  |FileSystem|本地文件系统	C:, D:|
  |Registry|Windows 注册表HKCU:|
  |Environment|环境变量$env:PATH|
  |Certificate|证书存储Cert:|
  |Function|当前会话中的函数|
  |Variable|当前定义的变量|
  |Alias|命令别名|
- 脚本与模块：组织与复用
  + 脚本（.ps1）:包含一系列命令、流程控制、函数等
  + 模块（.psm1 / .psd1）: 一个.psm1 文件（模块脚本）/一个包含 .psd1 清单的文件夹（高级模块）
  + 查看系统中所有模块：Get-Module -ListAvailable
- 执行策略（Execution Policy）
  + 它限制 .ps1 脚本文件的执行行为。 -> Get-ExecutionPolicy -List
  + Restricted:默认，不允许运行任何脚本 -> Get-ExecutionPolicy
  + RemoteSigned:允许本地脚本运行，远程脚本需签名
  + AllSigned:所有脚本都需签名
  + Bypass:无限制，不建议在生产中使用
- 远程管理（PowerShell Remoting）
  + 支持通过 WS-MAN（基于 WinRM） 或 SSH 协议，远程连接到另一台计算机并执行命令。
  + Enter-PSSession -ComputerName Server01
- Desired State Configuration（DSC）:声明式配置平台，用于定义系统的"理想状态"。
  + 编写配置文件（.ps1）
  + 编译成 MOF 文件
  + 使用 Push 或 Pull 模式部署
- 命令发现与帮助系统
  + Get-Command：列出所有命令
  + Get-Member：查看对象属性与方法-> Get-Process | Get-Member
